## 2. 学习行为分布 (studentStudyBehavior)

### 需要提取的字段

| 字段名 | 类型 | 说明 | 示例值 |
|--------|------|------|--------|
| `behaviorType` | String | 行为类型 | "主动学习" / "被动学习" |
| `startTime` | Long | 开始时间（时间戳） | 1745376270000 |
| `endTime` | Long | 结束时间（时间戳） | 1745376300000 |

### 需要计算的指标

- 被动学习总时长（秒）
- 主动学习占比（%）
- 被动学习占比（%）
- 加权知识留存率（%）

## 3. 回答建构分类 (solo)

### 需要提取的字段

原始数据为 `List<SoloAnswer>`，每个元素包含：

| 字段名 | 类型 | 说明 | 示例值 |
|--------|------|------|--------|
| `soloLevel` | Integer | SOLO分类等级 | 1-5 |
| `soloLevelName` | String | SOLO分类名称 | "前结构" / "单点结构" / "多点结构" / "关联结构" / "抽象拓展" |
| `count` | Integer | 该等级的回答数量 | 10 |

### SOLO等级说明

1. **前结构水平** - 躲避回答或重复问题
2. **单点结构水平** - 只有一个思路，回答简短（≤5字）
3. **多点结构水平** - 多个思路但无联系，有并列关系（约10字）
4. **关联结构水平** - 多个思路有机整合，形成结论
5. **抽象拓展结构水平** - 拓展至新领域，深化问题

### 需要计算的指标

- 各SOLO等级的回答数量
- 各SOLO等级的占比（%）
- SOLO总回答数

## 4. 应答时间 (studentAnswerClassification)

### 需要提取的字段

原始数据为 `List<StudentAnswerClassifyResult>`，每个元素包含：

| 字段名 | 类型 | 说明 | 示例值 |
|--------|------|------|--------|
| `sentenceId` | String | 句子ID | "abc123" |
| `startTime` | Long | 开始时间（时间戳） | 1745376270000 |
| `endTime` | Long | 结束时间（时间戳） | 1745376280000 |
| `studentAnswerType` | String | 回答类型 | 类型标识 |

### 需要计算的指标

根据 `endTime - startTime` 计算回答时长：

- **≤5秒** 回答数量和占比
- **5-15秒** 回答数量和占比
- **>15秒** 回答数量和占比
- 总回答数

## 6. 课堂流程重构 (courseProcessReengineering) - 教师页

### 需要提取的字段

#### 6.1 课前到课中链接 (preClassToInClassLink)

| 字段名 | 类型 | 说明 | 示例值 |
|--------|------|------|--------|
| `level` | String | 总体等级 | "初阶" / "进阶" / "中阶" / "高阶" |
| `subTasks` | Array | 子任务列表 | 包含3个子维度 |

**subTasks 包含的3个子维度：**

1. **学生完成自主学习**
   - `name`: "学生完成自主学习"
   - `score`: 分数/等级 (1-4)
   - `reason`: 分析原因文本

2. **教师课前提供学习资源**
   - `name`: "教师课前提供学习资源"
   - `score`: 分数/等级 (1-4)
   - `reason`: 分析原因文本

3. **教师分析学生学习数据**
   - `name`: "教师分析学生学习数据"
   - `score`: 分数/等级 (1-4)
   - `reason`: 分析原因文本

#### 6.2 课中到课后链接 (inClassToPostClassLink)

| 字段名 | 类型 | 说明 | 示例值 |
|--------|------|------|--------|
| `level` | String | 总体等级 | "初阶" / "进阶" / "中阶" / "高阶" |
| `subTasks` | Array | 子任务列表 | 包含1个子维度 |

**subTasks 包含的1个子维度：**

1. **布置课后学习任务**
   - `name`: "布置课后学习任务"
   - `score`: 分数/等级 (1-4)
   - `reason`: 分析原因文本

### 等级说明

- **初阶** (1分) - 无重构
- **进阶** (2分) - 初步重构
- **中阶** (3分) - 良好重构
- **高阶** (4分) - 高水平重构

## 7. 提问有效性 (questionScoreExplain) - 教师页

### 7.1 提问记录 (questionRecord)

| 字段名 | 类型 | 说明 | 示例值 |
|--------|------|------|--------|
| `questionList` | Array | 问题列表 | 见下方 |
| `questionCount` | Integer | 提问总数 | 20 |

**questionList 每个元素包含：**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `questionId` | String | 问题ID |
| `questionText` | String | 问题文本 |
| `questionTime` | Long | 提问时间戳 |

### 7.2 布鲁姆提问分类 (bloom)

原始数据为 `List<BloomProblem>`，每个元素包含：

| 字段名 | 类型 | 说明 | 示例值 |
|--------|------|------|--------|
| `bloomLevel` | Integer | 布鲁姆等级 | 1-6 |
| `bloomLevelName` | String | 等级名称 | "记忆" / "理解" / "应用" / "分析" / "评价" / "创造" |
| `count` | Integer | 该等级的问题数量 | 5 |

### 布鲁姆等级说明

1. **记忆** - 回忆事实和基本概念
2. **理解** - 解释想法或概念
3. **应用** - 在新情境中使用信息
4. **分析** - 分解信息，找出关联
5. **评价** - 判断和批判
6. **创造** - 产生新的想法

### 需要计算的指标

- 各布鲁姆等级的问题数量
- 各布鲁姆等级的占比（%）
- 高阶思维（分析/评价/创造）占比

### 7.3 教师理答分类 (teacherAppraisalClassification)

原始数据为 `List<TeacherAppraisalClassifyResult>`，每个元素包含：

| 字段名 | 类型 | 说明 | 示例值 |
|--------|------|------|--------|
| `appraisalType` | String | 理答类型 | "简单重复" / "追问" / "评价" / "引导" 等 |
| `count` | Integer | 该类型的次数 | 8 |

### 需要计算的指标

- 各理答类型的次数
- 各理答类型的占比（%）
- 高质量理答（追问、引导）占比

### 7.4 问答额外结果 (questionAnswerExtraResult)

| 字段名 | 类型 | 说明 | 示例值 |
|--------|------|------|--------|
| `totalAnswerCount` | Integer | 总回答次数 | 50 |
| `effectiveAnswerCount` | Integer | 有效回答次数 | 45 |

### 需要计算的指标

- 有效回答率 = `effectiveAnswerCount / totalAnswerCount`

### 7.5 提问有效性综合得分 (questionScoreExplain)

| 字段名 | 类型 | 说明 | 示例值 |
|--------|------|------|--------|
| `score` | Float | 提问有效性综合得分 | 71.0 |

**注意**: 这个 score 是基于布鲁姆分类、教师理答、学生SOLO等多维度计算出的综合分数

## 数据提取注意事项

1. **时间戳转换**: 所有时间戳为13位毫秒，需要转换为可读日期时间
2. **百分比计算**: 小数形式（如 0.726）需转换为百分比（72.6%）
3. **空值处理**: 部分接口可能返回 `null`，需做空值判断
4. **数组遍历**: 多个模块返回数组数据，需遍历后进行统计计算
5. **时长计算**: 基于 `startTime` 和 `endTime` 计算时长（秒）
